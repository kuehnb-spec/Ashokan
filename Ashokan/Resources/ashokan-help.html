Ashokan Help

A simple, fast, native editor for HTML and Markdown documents. This help file is itself an Ashokan document — a plain HTML file you could open anywhere.

What Ashokan is

Ashokan lets you edit HTML files the way a word processor edits documents — while keeping the file plain, open HTML that renders in any browser. Its core promise is the lossless round-trip: Ashokan never rewrites what you didn't edit. Everything outside <body> is preserved byte-for-byte; every element keeps its attributes; markup Ashokan doesn't model (scripts, SVG, widgets) is carried as protected islands that render but can't be mangled.

Opening and saving

  • Open .html, .htm, .md, and .markdown files (File > Open, drag onto the Dock icon, or double-click in Finder).
  • The Welcome window shows recent documents with live thumbnails, plus your workspace folders — add the folders where you keep documents and browse them as cards. Reopen it anytime with the clock toolbar button or ⇧⌘1.
  • Documents autosave in place; ⌘S saves explicitly. Multiple documents open as window tabs.
  • Markdown files are edited in the same WYSIWYG view and saved back as Markdown.

Images

Insert via ⇧⌘I, paste, or drag from Finder. Images embed as data URIs so the document stays one self-contained file. Click an image and drag the blue corner handle to resize; Format > Image sets inline/float/centered.

Tables

Insert from the Table menu or the table dropdown in the format bar. Tab moves between cells; add/delete rows and columns, merge/split cells, toggle the header row, and drag column borders to resize.

Printing and PDF export

File > Print… (⌘P) prints the document through the same paginated pipeline as PDF export: images, tables, and code blocks never split across pages, and headings stay with their text. File > Export as PDF… (⌥⌘P) produces the same result as a PDF file. Both warn you first if the document still has pending suggestions or comments.

Review: tracked changes and comments

Ashokan records suggestions as standard HTML — <ins> and <del> elements — so a document with pending edits still renders as redlines in any browser. Comments are <mark title="…"> elements, readable on hover anywhere.

  • Suggest mode — toggle Suggest in the format bar (⇧⌘E). Your typing, deletions, and pastes become tracked changes with your name on them.
  • Review bar — summon it anytime with the checklist button in the toolbar (or View > Review Bar, ⇧⌘R); it also appears automatically whenever the document has pending changes or comments. The View menu lists every bar — Format, Review, Comments in Margin, Source, Status — with checkmarks, and the matching toolbar buttons glow with your accent color while their bar is showing. Navigate (⌥⌘[ / ⌥⌘]), accept/reject one at a time, or use the Accept All / Reject All menus to resolve everything, just the selection, or everything by one author (you vs. the AI).
  • Preview modes — the review bar's Markup / Final / Original control shows the document as-is, as if every suggestion were accepted, or as if all were rejected. View-only: an orange banner reminds you the markup is still in the file, and editing pauses until you return to Markup.
  • Task lists — GFM checkboxes (- [ ] in Markdown, checkbox inputs in HTML) are real and clickable; toggling writes back to the file.
  • Two writers, one file — if an agent rewrites the open document on disk, Ashokan reloads it automatically (status-bar notice); if you also had unsaved edits, it asks which version wins.
  • Hover a change — a ✓/✕ chip appears; accept or reject in place.
  • Comments — select text, Add Comment (⌥⌘M). Click a yellow highlight to read/edit/remove it. Show All lays every comment out as margin cards with Edit/Remove actions.

AI review — local models

Review > AI Review with Local Model… (⌥⌘R) sends the document text to your own Ollama server at localhost:11434 — nothing leaves your Mac. The model's suggestions arrive as tracked changes with the model named as author; accept or reject each like any other reviewer's work.

Working with AI agents (Claude Code, Codex, etc.)

Review > Add Agent Instructions… embeds an invisible instruction block (an HTML comment) that teaches any AI agent the tracked-changes protocol. Then tell your agent: "Review ~/path/to/file.html and follow its embedded instructions." Its edits arrive as redlines in Ashokan.

Agents can also read the machine-oriented guide bundled with the app: Help > Agent Guide, or point them at /Applications/Ashokan.app/Contents/Resources/AGENTS.md.

Direct connection (MCP)

Review > Allow Agents to Connect (MCP Server) lets agents talk to Ashokan directly: they can list and read your open documents and propose edits — which always arrive as tracked changes under the agent's name, never silent rewrites. Then Review > Copy Agent Setup Command and paste it in a terminal to connect Claude Code. Off by default; when off, nothing runs. When on, it listens only on this Mac (127.0.0.1, token-protected), and normal editing is unaffected.

Troubleshooting

  • A widget or styled block won't edit: it's a protected island (markup Ashokan doesn't model). Edit it in the source pane (⌘/).
  • Formatting buttons do nothing: you're in Suggest mode — formatting changes can't yet be tracked, so they're blocked rather than applied silently. Toggle Suggest off.
  • AI Review says Ollama isn't reachable: start Ollama and confirm localhost:11434 answers; at least one model must be pulled.
  • This help file is locked: it lives inside the app bundle. Duplicate it (File > Save As…) to experiment on it.
